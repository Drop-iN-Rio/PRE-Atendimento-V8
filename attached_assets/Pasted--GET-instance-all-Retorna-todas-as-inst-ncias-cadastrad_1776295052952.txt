Aqui estão os detalhes completos do endpoint POST /instance/create da Evolution Go API:
Método e URL

Método: POST
Endpoint: /instance/create
URL exemplo: http://localhost:8080/instance/create

Descrição
Cria uma nova instância WhatsApp no servidor. Após criar, a instância começa desconectada e você precisa conectar usando QR Code ou pairing code.
Headers (obrigatórios)

Content-Type: application/json
apikey: SUA_CHAVE_API_GLOBAL

Body (JSON) – Parâmetros principais

Exemplo completo de Requisição (cURL)
Bashcurl --request POST \
  --url http://localhost:8080/instance/create \
  --header 'Content-Type: application/json' \
  --header 'apikey: SUA_CHAVE_API_AQUI' \
  --data '{
    "instanceName": "minha-instancia",
    "qrcode": true,
    "integration": "WHATSAPP-BAILEYS",
    "token": "",
    "number": "",
    "webhook": {
      "enabled": true,
      "url": "https://exemplo.com/webhook",
      "events": ["message", "status"]
    }
  }'

Após criar, use o endpoint /instance/{instanceName}/qrcode ou /instance/connect para conectar.
Observações importantes

O instanceName deve ser único.
Se qrcode: true, a resposta já pode vir com o QR Code em base64.
A instância é criada desconectada (connected: false).
Para conectar depois: use POST /instance/connect ou peça o QR Code.
